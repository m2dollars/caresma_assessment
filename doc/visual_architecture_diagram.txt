DATA FLOW SUMMARY:
1. User speaks → Audio captured by frontend
2. Audio sent to FastAPI backend → Queued in Redis job queue
3. Celery worker processes speech-to-text (Deepgram) → Text transcription
4. Text queued for cognitive assessment → OpenAI GPT-4 analysis
5. Assessment results queued for storage → PostgreSQL database
6. Calendar scheduling queued → Google Calendar API integration
7. Real-time updates sent to frontend via WebSocket
8. Family/Provider notifications queued → Multi-channel delivery

KEY ARCHITECTURAL PRINCIPLES:
• Voice-first design for elderly users
• Privacy-first data handling
• Modular AI assessment engines
• Healthcare compliance (HIPAA, SOC 2)
• Scalable microservices architecture
• Redis job queue for efficient LLM processing
• Asynchronous task processing with Celery
• Real-time processing capabilities
• Multi-channel notification system
